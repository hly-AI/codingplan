# CodingPlan 需求补全

为 CodingPlan 工作流 Step 2 提供需求文档结构与模板。

## 文档结构

```markdown
# 需求：{需求名}

## 1. 目标描述
- 业务目标
- 用户价值
- 成功标准

## 2. 功能范围
- 功能点列表
- 优先级（P0/P1/P2）
- 不包含范围

## 3. 功能详情
### 3.1 功能 A
- 描述
- 用户故事（可选）：作为 [角色]，我希望 [能力]，以便 [价值]
- 验收标准
  - [ ] 条件 1
  - [ ] 条件 2

### 3.2 功能 B
...

## 4. 非功能性需求
- 性能：响应时间、并发
- 安全：认证、权限、数据
- 可用性：兼容性、可访问性

## 5. 约束条件
- 技术栈
- 接口约束
- 依赖限制
```

## 补全原则

- 目标明确、可衡量
- 功能范围边界清晰
- 验收标准可验证
- 不确定内容写入 uncertain/
