# CodingPlan 代码实现

为 CodingPlan 工作流 Step 5 提供实现检查清单与指引。

## Plan 模式步骤

1. 拆解实现步骤（按模块或功能点）
2. 确认依赖与顺序
3. 逐步编码，每步验证
4. 完成后再进入下一模块

## 实现检查清单

```
- [ ] 遵循详细设计中的模块与接口定义
- [ ] 遵守项目 .cursor/rules（安全、代码质量、API、UI 等）
- [ ] 若指定 scope，仅修改 scope 目录内
- [ ] 若有 Figma，按设计稿实现 UI
- [ ] 若涉及多端，各端均已实现
- [ ] 无硬编码敏感信息
- [ ] 公共 API 有注释
```

## 实现顺序建议

1. 数据结构/模型
2. 核心逻辑/服务
3. 接口/API
4. UI 组件（若涉及）
5. 集成与联调
